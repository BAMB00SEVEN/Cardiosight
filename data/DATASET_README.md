# Dataset Notes

`sample_features_labeled.csv` in this folder is **synthetic** — generated by
`ml/generate_synthetic_dataset.py` purely so the training/inference pipeline
can be run and tested end-to-end without waiting on real hardware or a
downloaded dataset. **Do not report its accuracy numbers as project results.**

## Recommended real dataset: MIT-BIH Arrhythmia Database

For real, citable results, use the **MIT-BIH Arrhythmia Database** on
PhysioNet (free, widely used in academic ECG-ML papers):

- https://physionet.org/content/mitdb/1.0.0/
- 48 half-hour two-lead ECG recordings, annotated beat-by-beat by
  cardiologists (Normal, PVC, APB, AFib-related rhythms, etc.)

### Suggested workflow to build your real `features_labeled.csv`

1. Download the MIT-BIH records (`.dat`, `.hea`, `.atr` files) via the
   `wfdb` Python package:
   ```bash
   pip install wfdb
   ```
2. For each record, load the signal + annotations:
   ```python
   import wfdb
   record = wfdb.rdrecord('100', pn_dir='mitdb')
   annotation = wfdb.rdann('100', 'atr', pn_dir='mitdb')
   ```
3. For each annotated beat, extract a window around the beat, run it through
   `ml/filtering.py` (`clean_ecg`), then `ml/feature_extraction.py`
   (`extract_feature_vector`) to get the same feature columns used in
   `train_model.py`.
4. Map MIT-BIH's annotation symbols to your project's disease-indication
   classes, e.g.:
   - `N` → Normal
   - `V` → PVC (Premature Ventricular Contraction)
   - `A` → APB (Atrial Premature Beat)
   - AFib episodes (from rhythm annotations) → AFib
   - everything else → Other
5. Save the resulting rows as `data/features_labeled.csv` with the same
   columns as `sample_features_labeled.csv`, then run:
   ```bash
   python3 train_model.py --data ../data/features_labeled.csv
   ```

### If you're also collecting your own hardware data

Log raw AD8232/ADS1292R output for known-normal subjects (with appropriate
ethics/consent if this is a formal study) to validate that your filtering +
feature extraction produces sane heart-rate values before trusting the
classifier on real-world data. Real-world single-lead hobbyist ECG data is
**not** a substitute for a diagnostic-grade dataset — frame your project's
disease-indication output as a **screening aid / educational demonstrator**,
not a certified diagnostic tool, in your report.
